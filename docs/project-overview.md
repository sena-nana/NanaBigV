# 项目概览

本页是文档站内的 README 镜像，用于在 VitePress 站内阅读 NaNaBigV 的产品目标、MVP 闭环和 V1 边界。仓库根目录的 `README.md` 仍然是对外入口文件。

## NaNaBigV 是什么

NaNaBigV 是一个面向主播的本地桌面控制台，用于 AI 观众氛围模拟、直播演练、弹幕灵感生成和直播控场提词。

系统以主播语音转文本为主输入，结合 Echo-Live 输入和直播画面识别摘要作为增强上下文，请求 OpenAI 兼容格式的 provider，生成可审核的观众反应候选，并通过本地队列与改造后的 blivechat 前端呈现互动效果。

## MVP 页面

- 工作台：快速开始、继续上次方案、进入演练/中控台和查看关键状态。
- 新建直播：4 步配置直播类型、直播信息、AI 观众氛围和启动前检查。
- 直播中控台：集中展示主播语音优先链路、Echo-Live 状态、候选弹幕、主播提词和控场按钮。
- AI 观众：管理观众组、频率倾向、语言风格、边界规则和记忆范围。
- 话题库：管理今日大纲、话题卡、推荐弹幕、主播接话和梗库。
- 安全设置：管理输出模式、基础安全、质量过滤和频率上限。
- 弹幕记录：查看本地队列和生成记录，按状态、观众组、触发原因和风险筛选。

## V1 目标

- 围绕一个主播、一个直播间、本地单操作者完成最小闭环。
- 以主播语音为必需输入，支持 Echo-Live 与视觉上下文作为可选增强。
- 支持 4 类互动输出：`danmaku`、`gift`、`super_chat`、`membership`。
- 用结构化长期记忆保存主播设定、历史互动事实、观众画像和场次摘要，持续影响后续生成。
- 通过 Tauri 本地 store 保存直播方案、观众组、话题库、安全设置和生成记录。

## 非目标

- 多直播间并发管理。
- 云端编排、多租户或团队协作后台。
- 真实平台运营后台、账号体系或平台级权限管理。
- OBS/VTS 深度集成、模板市场、素材库或完整直播复盘时间线。
- 房管、抽奖、活动脚本、运营战役编排等复杂活动系统。
- 首版即重度插件化或厂商私有协议适配。
