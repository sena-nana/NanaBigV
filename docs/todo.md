- [x] 保留旧路由兼容重定向：`/danmaku -> /live`、`/audience -> /audience-groups`、`/quota -> /settings?tab=provider`、`/review -> /danmaku-records`。

- [x] 本地配置覆盖直播方案、观众组、话题卡、今日大纲、梗库、安全设置和生成记录。

- [x] 冷场救急按钮通过现有本地队列生成候选，不绕过安全过滤和投递开关。

## 已完成的基础能力

- [x] OpenAI 兼容 Provider 配置、模型列表、连通性测试和结构化请求。
- [x] 主播语音文本接入统一上下文窗口。
- [x] Echo-Live WebSocket 文本输入作为增强上下文。
- [x] 视觉上下文保留统一事件模型和状态入口。
- [x] `MemoryStore` 本地持久化，覆盖主播设定、长期互动事实、观众画像和场次摘要 4 层。
- [x] 记忆检索与受控写回入口，写回结果区分 accepted、quarantined、rejected。
- [x] `AudiencePlanner` 根据上下文窗口、记忆观众、配置观众组、队列压力和安全通道生成互动意图。
- [x] 本地 `BlivechatEventQueue` 记录 `enqueue`、`deliver`、`throttle` 协议动作。
- [x] 本地 mock 数据源支持启动、暂停、单步和重置，用于验证上下文、planner、provider、队列和记忆写回链路。
