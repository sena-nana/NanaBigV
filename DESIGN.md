# 桌面应用模板视觉设计标准

## 0. 适用范围

本文定义模板应用的视觉语言,用于指导桌面端界面、后续业务页面和 UI 改动评审。

模板面向工程工具类桌面应用。它不是营销站点、通用聊天应用或装饰性仪表盘。界面应帮助用户理解当前位置、当前状态、可执行操作和待处理决策。

## 1. 设计原则

- 克制、清晰、可扫描。
- 信息密度适合重复工作,但不制造噪音。
- 默认保持安静,只在用户必须行动时提高可见度。
- 主内容 > 当前状态 > 过程信息 / 辅助操作。

每个主界面都应让用户快速回答:

- 我现在在哪里?
- 当前页面状态是什么?
- 下一步是否需要我操作?

## 2. 布局系统

- 标题栏:应用名、侧栏开关、窗口控制。
- 左侧栏:全局操作、导航树、左下角按钮区和连接状态。
- 主工作区:当前页面内容。
- 设置页:使用设置专用侧栏承载分类导航,主区只显示当前设置分类。

避免落地页式构图、hero 区块、超大标题、装饰面板和营销式卡片流。卡片只用于独立信息组、重复项、弹窗和工具容器,不要嵌套卡片。

## 3. 色彩令牌

优先使用 `src/styles.css` 中已有 CSS 变量。

| 令牌 | 用途 |
| --- | --- |
| `--bg` | 主工作区表面 |
| `--bg-elev` | 标题栏、侧栏、卡片、浮层 |
| `--bg-subtle` | 输入框、代码块、chip、非激活列表表面 |
| `--bg-hover` | 行和轻量控件 hover 背景 |
| `--bg-active` | 激活行、选中 tab、按下状态 |
| `--border-soft` | 低强调分隔线和内部边界 |
| `--border` | 标准组件边界 |
| `--border-strong` | 浮层、激活输入、强交互边界 |
| `--text` | 主文本 |
| `--text-muted` | 次要标签、元信息、非激活控件 |
| `--text-faint` | 占位、路径、弱提示 |
| `--accent` | 当前项、主操作、焦点 |
| `--accent-strong` | 强主操作、需要比 `--accent` 更高对比的强调 |
| `--accent-soft` | 选中底色、轻量主色 hover、连接正常底色 |
| `--accent-text` | 主色按钮上的文字 |
| `--ok` / `--warn` / `--err` | 成功、警告、错误状态 |
| `--ok-soft` / `--warn-soft` / `--err-soft` | 状态底色、危险 hover 或确认态底色 |

soft 令牌只用于状态底色、选中底色和危险 hover,不要扩大成页面背景、装饰块或大面积铺色。

## 4. 样式组织

- `src/styles.css` 只保留主题令牌、reset、字体、滚动条、selection、基础表单控件和全局工具类。
- 桌面壳布局放在 `src/styles/shell.css`,由壳层入口显式导入。
- 页面通用结构放在 `src/styles/page.css`,例如 `.page-header`、`.card`、`.kv`。
- 组件私有样式保留在对应 SFC 的 scoped style 中,避免把局部类名继续堆进全局样式。
- 新增样式先判断归属:跨页面复用才进入页面通用样式,壳层专属才进入 shell 样式,单组件行为留在组件内。

## 5. 组件语言

- 熟悉工具优先使用图标按钮:折叠、搜索、设置、扩展、窗口控制。
- 有可用图标时优先使用 `lucide-vue-next`。
- 需要明确语义的命令使用文字按钮。
- 行需要支持 hover、active、muted 和 disabled 状态。
- 文案短、直、可操作;按钮写动作,状态写事实,提示写影响。
- hover、active 反馈优先改变背景和文字颜色,不要依靠放大、位移、强阴影或高饱和装饰色。

## 6. 桌面壳

- 标题栏高度固定为 36px,整体保持紧凑、安静。
- 标题栏使用左控件区、居中品牌、右窗口控制三段结构;品牌文本居中,超长时省略。
- 标题栏的侧栏按钮和窗口按钮使用 28px 图标按钮,默认透明背景,hover 时使用 `--bg-hover`。
- 关闭等危险窗口操作只在 hover 时使用 `--err-soft` 和 `--err`。
- 主工作区使用 20px 24px 内边距,背景为 `--bg`,左上角用 10px 圆角承接侧栏区域。
- 壳层 resize 时禁用主区和侧栏 pointer events,避免拖拽过程中误触内容。

## 7. 侧边栏

- 侧边栏宽度可拖拽并持久化,双击拖拽线恢复默认宽度。
- 折叠时侧边栏淡出并禁用拖拽线,离开特殊模式后恢复用户偏好。
- 区域之间依靠 gap 留白,不额外加装饰分隔线。
- 顶部全局按钮和区块工具保持紧凑尺寸,hover 时反馈清晰但克制。
- 区块标题高度 24px,使用 11px 大写弱标签;区块工具默认隐藏,hover 或 focus-within 时出现。
- 侧栏 action 高度 30px,导航树行高度 28px,图标工具按钮 22px,保证重复扫描时稳定。
- 侧栏行文本必须 `min-width: 0` 并使用省略号处理溢出。

### 左下角按钮区

- 左下角 footer 是辅助区域,默认应自动弱化。
- footer 按钮使用 26px 尺寸和 5px 圆角,按钮之间只保留 2px 紧凑间距。
- `.sb-footer__btn` 默认低透明度;当 footer hover、focus-within 或按钮 active 时恢复完全可见。
- 连接胶囊 `.sb-conn` 默认轻度弱化,hover/focus 时增强。
- `ok/warn/error/probing` 状态只表达真实状态,不要作为装饰色使用。

## 8. 页面、卡片和浮层

- 页面标题使用 18px/600,说明文字使用 13px 和 `--text-muted`;页面头部只做定位和状态说明,不要变成 hero。
- 卡片圆角 8px,内边距 14px 16px,边框使用 `--border`,只承载独立信息组、重复项、弹窗或工具容器。
- 卡片标题使用 13px/600、`--text-muted`、大写和轻微字距,表达分组而不是制造大标题。
- key-value 列表左右分布,值允许 `overflow-wrap: anywhere`,避免路径、版本号或长标识撑破布局。
- 菜单、下拉和上下文菜单使用 `--bg-elev`、边框令牌、4px 内边距和克制阴影;菜单项高度保持 28px 左右。
- 弹窗宽度使用视口约束,确认类危险操作只在危险项 hover、pending 或确认态使用错误色。

## 9. 动效和状态

- 常规 hover、active、边框和文字颜色过渡约 0.12s,保持及时反馈。
- 侧栏折叠使用 `cubic-bezier(0.2, 0.8, 0.2, 1)`,同步处理 opacity、transform 和 visibility。
- 折叠状态下隐藏拖拽线并禁用 pointer events;resize 状态下移除不必要 transition。
- 弹窗进入退出可以使用轻微透明度和位移,但必须支持 `prefers-reduced-motion: reduce`。
- loading、empty、disabled 状态不能改变控件固定尺寸或造成布局跳动。

## 10. 设置页

- 设置页使用专用侧栏展示分类,主区只显示当前分类内容。
- 返回按钮在设置侧栏顶部,回到进入设置前的主窗口路由。
- 分类行使用图标 + 短标签,支持 hover 和 active 状态。
- 深浅主题都必须保持可读,文本不能溢出按钮、chip、行或卡片。

## 11. 评审清单

- 界面仍然像克制的工程工具。
- 主内容强于状态和辅助控件。
- 新颜色来自现有令牌。
- 新样式放在正确归属:全局、shell、page 或组件 scoped。
- 深浅主题都可读。
- 控件在 hover、active、loading、empty 状态下尺寸稳定。
- 标题栏、侧栏和页面基础组件保持模板既有紧凑尺寸。
- 动效不抢注意力,并尊重 reduced motion。
- 浮层、菜单和危险状态使用克制边框、阴影和状态色。
- 侧边栏 footer 默认弱化,hover/focus 后可读。
- 设置侧栏 active 项明确,返回路径符合预期。
