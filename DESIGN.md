# NaNaBigV 视觉设计标准

## 0. 适用范围

本文定义 NaNaBigV 的桌面端视觉语言，用于指导直播中控台、配置页、调试面板和后续业务页面的设计与评审。

NaNaBigV 是工程工具，不是营销站点，也不是通用聊天应用。界面应帮助操作者快速理解当前直播状态、上下文输入、模拟行为和系统异常。

## 1. 设计原则

- 克制、清晰、可扫描。
- 先看直播状态，再看模拟结果，最后看调试细节。
- 默认保持安静，只在需要人工处理时提高可见度。
- 页面必须能快速回答 3 个问题：现在在看什么、系统刚做了什么、接下来是否需要我介入。

信息层级固定为：

- 主内容：直播上下文、互动流、关键配置。
- 当前状态：连接状态、输入源状态、模型状态、记忆状态。
- 过程信息：日志、调试、原始事件、失败细节。

## 2. 布局系统

- 标题栏：应用名、全局状态、窗口控制。
- 左侧栏：导航、直播间视图切换、调试入口、设置入口。
- 主工作区：互动流、输入概览、控制面板或详情视图。
- 次级面板：用于调试、记忆详情、原始事件、策略解释，不应抢占主工作区。

避免营销式 hero、装饰性大卡片和与直播控制无关的统计面板堆砌。

## 3. 页面类型

- 工作台：显示快速开始、继续上次方案、连接状态、当前模型和最近复盘摘要。
- 直播中控台：显示主播语音优先链路、候选弹幕、主播提词和控场按钮。
- AI 观众页：管理观众组、频率倾向、语言风格、边界规则和记忆范围。
- 话题库页：管理今日大纲、话题卡、推荐弹幕和梗库。
- 安全设置页：管理输出模式、基础安全、质量过滤和频率上限。
- 弹幕记录页：查看本地队列与生成记录，按状态和风险筛选。
- Provider 配置页：管理 OpenAI 兼容接口、模型参数和连通性。
- 调试页：查看输入事件、提示组装、生成结果、频率控制和投递日志。
- 设置页：只放全局偏好、主题和本地行为。

每个页面都应有明确主任务，不把配置、调试和展示混在一个大面板里。

## 4. 色彩与令牌

优先使用 `src/styles.css` 中已有 CSS 变量。

- `--bg` / `--bg-elev`：主背景和浮层。
- `--text` / `--text-muted` / `--text-faint`：主文本、次文本、弱提示。
- `--accent` / `--accent-soft` / `--accent-text`：当前项、主操作、选中态。
- `--ok` / `--warn` / `--err`：真实状态，不做装饰色滥用。

状态色只表达事实：

- `ok`：输入源可用、provider 可用、投递正常。
- `warn`：上下文缺失、回退运行、节奏受限。
- `err`：调用失败、解析失败、投递失败、记忆写回失败。

不要把大面积彩色背景当作“直播氛围”。真实感来自节奏与信息组织，不来自高饱和装饰。

## 5. 组件语言

- 以行、列表、卡片、标签、轻量图表为主。
- 优先使用图标按钮和短动词按钮，不写长句式 CTA。
- 互动流中的每条记录要能一眼看出类型、来源、角色和时间。
- 调试信息默认折叠或弱化，避免与主互动流争抢注意力。
- 同一类状态在所有页面使用同一种标签和颜色语义。

## 6. 直播互动流

- 互动流是核心视觉区域，应优先于配置面板。
- 互动项需稳定显示类型、观众身份、文案、金额或等级、触发原因。
- 弹幕、礼物、SC、舰长的样式可以有差异，但结构和节奏必须统一。
- 高频弹幕状态下不能因为动画、阴影或尺寸变化造成阅读抖动。
- 当输出被节流、拒绝或回退时，应有轻量可追踪的状态标记。

## 7. 输入与上下文展示

- 主播语音是主输入，视觉上优先级最高。
- Echo-Live 输入和视觉摘要作为增强源，默认弱于主播语音，但状态必须可见。
- 输入卡片既要展示“最近收到什么”，也要展示“当前缺什么”。
- 长文本摘要默认截断，展开后保留结构，不做聊天气泡式排版。

## 8. 记忆与策略视图

- 主播设定、长期事实、观众画像、场次摘要分开显示，不混成一个大列表。
- 记忆项应有来源、更新时间和可信度或写回依据。
- 策略解释面板用于回答“为什么发这条互动”，而不是复述原始模型输出。
- 污染风险、冲突事实和低置信度内容必须可见。

## 9. 动效与反馈

- 常规 hover、active、边框和文字颜色过渡保持在约 0.12s。
- 互动流允许轻量进入动画，但不能影响高频扫描。
- loading、empty、disabled 状态不能改变控件尺寸。
- 支持 `prefers-reduced-motion: reduce`。

## 10. 评审清单

- 仍然像克制的工程控制台，而不是直播装饰页。
- 主内容优先于日志和辅助操作。
- 主播语音、增强上下文、生成结果、投递结果的层次清楚。
- 记忆、策略、调试信息已弱化但可追踪。
- 深浅主题可读，状态色表达真实系统状态。
- 高频互动下列表、时间线和卡片不会跳动或难以扫描。
